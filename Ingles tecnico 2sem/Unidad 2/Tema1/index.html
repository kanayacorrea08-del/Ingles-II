Introducción

En programación no solo importa escribir código, también comunicar procesos de forma clara. Desde instalar software, ejecutar un programa, configurar un entorno o corregir errores, es necesario explicar cada paso con orden y coherencia. Las sequencing expressions (first, then, next, after that, finally) ayudan a estructurar instrucciones técnicas, mejorar la comprensión y facilitar el trabajo colaborativo.

En esta guía aprenderás a identificar y usar estas expresiones para describir procesos reales del entorno tecnológico con claridad, precisión y sentido lógico.

Objetivos de aprendizaje

  • ✓ Aplicar expresiones secuenciales para explicar procesos técnicos en programación (instalación, ejecución, depuración y desarrollo de funcionalidades) organizando ideas de forma lógica y coherente.
  • ✓ Organizar instrucciones técnicas con conectores como first, then, next, after that, finally.
  • ✓ Comunicar procedimientos de forma clara, precisa y efectiva en contextos reales del entorno tecnológico.

Contenido

Antes de aprender sequencing expressions, piensa en una actividad técnica que hayas realizado recientemente (instalar software, escribir código, ejecutar un programa o corregir un error).

Responde en tu cuaderno: ¿Qué pasos sigues? ¿Cómo explicarías ese proceso a otra persona?

¿Qué son las sequencing expressions?

  • ✓ Son palabras o frases para indicar el orden de los pasos en un proceso.
  • ✓ Permiten explicar procesos paso a paso y dar instrucciones claras.
  • ✓ Organizan ideas de forma lógica y mejoran la comunicación profesional.

Ahora observa

Lee estas instrucciones:

Open the code editor. Write the program. Run the code. Check the output.

Pregunta: ¿Cuál es el problema? Las acciones son correctas, pero no están conectadas con palabras que indiquen orden.

Reflect: Why is it important to explain a process step by step in programming?

text 
First, open the code editor.
Then, write the program.
Next, run the code.
Finally, check the output.

Think about it: En programación no solo importa qué haces, sino también cómo lo explicas.

Ejemplos básicos y otras expresiones

First (primero), Then (luego), Next (después), After that (después de eso), Finally (finalmente).

También puedes usar: Before that, Afterwards, At the same time, In the end.

¿Por qué es importante? Organiza información, mejora claridad, facilita trabajo en equipo y permite comunicar procedimientos técnicos correctamente.

text 
First, open the application.
Then, install the software.
Next, configure the settings.
After that, run the program.
Finally, check the results.

Sequencing in Technical Instructions

Estas expresiones ayudan a que un proceso técnico sea claro y fácil de entender, especialmente en documentación y trabajo colaborativo.

Conectores de secuencia
1. Conectores para iniciar un proceso: ▼
First, First of all, To begin with
2. Conectores para continuar ▼
Then, Next, After that, Subsequently
3. Conectores para acciones simultáneas ▼
Meanwhile, At the same time
4. Conectores de cierre ▼
Finally, In the end, Lastly

Ejemplo 3: How to run a code

Situación: explica la secuencia para ejecutar un programa y verificar resultados.

text 
1. First, open the code editor.
2. Next, write the program.
3. Then, run the code.
4. After that, review the output.
5. Finally, fix errors if needed and save the final version.

Buenas prácticas al usar sequencing expressions:

  • ✓ Usa un conector por paso para mantener la secuencia clara.
  • ✓ Prefiere verbos técnicos precisos: connect, verify, restart, calibrate.
  • ✓ Cierra cada procedimiento con una acción de verificación final.
